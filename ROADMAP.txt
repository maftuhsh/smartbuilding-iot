# ROADMAP PROJECT PBL IoT

## Smart Building Berbasis IoT untuk Monitoring, Efisiensi Energi, dan Keamanan Ruangan

---

# TUJUAN PROJECT

Membuat sistem Smart Building berbasis IoT yang mampu:

* Monitoring kondisi ruangan secara realtime
* Menghemat energi secara otomatis
* Mendeteksi kondisi berbahaya
* Mengontrol perangkat dari jarak jauh
* Menampilkan dashboard online yang dapat diakses dari mana saja

---

# FITUR UTAMA SISTEM

## Monitoring

* Suhu ruangan
* Kelembapan ruangan
* Intensitas cahaya
* Deteksi asap/gas
* Deteksi keberadaan manusia

## Automation

* Lampu otomatis
* Fan otomatis
* Alarm asap/gas
* Eco mode hemat energi

## Dashboard

* Monitoring realtime
* Grafik data sensor
* Kontrol perangkat
* Mode auto/manual

---

# PHASE 3 — MEMBUAT SIMULASI WOKWI

## Tujuan

Membangun sistem dasar dan automation logic.

---

## STEP 1 — Setup ESP32

### Yang Dikerjakan

* Menambahkan ESP32 di Wokwi
* Menghubungkan ESP32 ke WiFi
* Testing Serial Monitor

### Output

ESP32 berhasil terkoneksi WiFi.

---

## STEP 2 — Menambahkan Sensor Satu per Satu

### Urutan Sensor

1. DHT22
2. LDR
3. PIR Motion Sensor
4. MQ2 Smoke Sensor

### Yang Dikerjakan

* Membaca data sensor
* Testing sensor
* Kalibrasi nilai sensor

### Output

Semua sensor dapat dibaca ESP32.

---

## STEP 3 — Menambahkan Actuator

### Urutan

1. Relay Lamp
2. Fan
3. Servo
4. Buzzer

### Yang Dikerjakan

* Testing ON/OFF actuator
* Menghubungkan actuator dengan sensor

### Output

Actuator dapat dikontrol ESP32.

---

## STEP 4 — Membuat Automation Logic

### Contoh Logic

IF suhu > 30°C
→ fan ON

IF cahaya rendah dan ada orang
→ lampu ON

IF tidak ada orang
→ lampu OFF

IF asap terdeteksi
→ buzzer ON

### Output

Sistem otomatis berjalan dengan baik.

---

# PHASE 4 — INTEGRASI MQTT

## Tujuan

Mengirim data sensor secara realtime.

## Yang Dikerjakan

* ESP32 publish data MQTT
* Membuat topic MQTT
* Testing komunikasi realtime

## Contoh Topic MQTT

smartbuilding/temperature
smartbuilding/humidity
smartbuilding/motion
smartbuilding/smoke
smartbuilding/device

## Tools

* HiveMQ Broker
* MQTT Explorer

## Output

Data sensor berhasil dikirim realtime.

---

# PHASE 5 — FIREBASE REALTIME DATABASE

## Tujuan

Menyimpan data sensor di cloud.

## Yang Dikerjakan

* Membuat Firebase Project
* Menghubungkan ESP32 ke Firebase
* Mengirim data sensor ke database

## Contoh Data

{
"temperature": 29,
"humidity": 70,
"motion": true,
"smoke": false
}

## Output

Data sensor tersimpan realtime di cloud.
